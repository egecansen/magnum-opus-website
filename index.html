How it works

Four phases.
Zero handoffs.

Opus runs an unbroken loop: it discovers what your product became, composes the tests it now needs, executes them in parallel across browsers and viewports, and quietly repairs itself when the surface shifts. You only hear from it when something a human should decide.

  1. 01 — Map

    Discover the surface area

    Agents crawl your product, infer journeys, build the live application graph.

  2. 02 — Compose

    Write the full variant set

    Happy path, edges, error states, mobile, data lifecycles — composed per journey, never duplicated.

  3. 03 — Execute

    Run continuously, in parallel

    Chromium, WebKit, Firefox · desktop and mobile · on every deploy and every hour in between.

  4. 04 — Heal

    Self-repair on drift

    When DOM, copy, or flow changes, Opus rewrites the affected steps and re-verifies — no PR needed.
